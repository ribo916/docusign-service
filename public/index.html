Overview

This sample REPL demos the DocuSign embedded signing process embedded in a generic consumer web application


1 - Create Envelope
  • Envelope created with documents, recipients, and webhook configs
  • Recipients are created with a signing order
  • Recipients are each configured to be embedded vs remote
  • Webhook notifications are sent via DocuSign Connect
2 - Create Recipient View
  • Link is 300 second TTL, one time use
  • Link can be requested numerous times
  • Link request will fail (with proper error) is link requested out of order
  • Can configure ping and ping frequency to maintain the application state during redirect
  • Return redirect URL configured when creating link
  • Query parameters passed back via redirect (e.g. session)
Misc Notes
  • DocuSign STRONGLY recommends Redirect over iFrame
  • Plan is to use PDF signing
